# Contributing to Portfolio\n\nThank you for your interest in contributing! This document provides guidelines and instructions for contributing to this project.\n\n## 🎯 Getting Started\n\n1. Fork the repository\n2. Clone your fork:\n   ```bash\n   git clone https://github.com/YOUR_USERNAME/portfolio.git\n   cd portfolio\n   ```\n3. Add upstream remote:\n   ```bash\n   git remote add upstream https://github.com/Forthy25/portfolio.git\n   ```\n4. Install dependencies:\n   ```bash\n   npm install\n   ```\n\n## 🚀 Development Workflow\n\n1. Create a feature branch:\n   ```bash\n   git checkout -b feature/your-feature-name\n   ```\n\n2. Make your changes and test them:\n   ```bash\n   npm run dev\n   npm run lint\n   ```\n\n3. Commit with clear messages:\n   ```bash\n   git commit -m \"fix: brief description of changes\"\n   ```\n\n4. Push to your fork:\n   ```bash\n   git push origin feature/your-feature-name\n   ```\n\n5. Create a Pull Request with a clear description\n\n## 📝 Commit Message Guidelines\n\nUse conventional commits:\n- `feat:` - New feature\n- `fix:` - Bug fix\n- `docs:` - Documentation\n- `style:` - Code style changes (formatting, semicolons, etc.)\n- `refactor:` - Code refactoring\n- `perf:` - Performance improvements\n- `test:` - Test changes\n\nExample:\n```bash\ngit commit -m \"feat: add dark mode toggle to header\"\n```\n\n## 🎨 Code Style\n\n- Follow the existing code structure\n- Use React functional components with hooks\n- Use meaningful variable and function names\n- Keep components focused and single-responsibility\n- Format code with proper indentation\n\n## 🧪 Testing\n\n- Test your changes locally:\n  ```bash\n  npm run dev\n  ```\n- Check responsiveness on mobile and desktop\n- Verify all links work correctly\n- Run linter:\n  ```bash\n  npm run lint\n  ```\n\n## 📦 Supported Browsers\n\n- Chrome (latest)\n- Firefox (latest)\n- Safari (latest)\n- Edge (latest)\n\n## 🐛 Reporting Bugs\n\nWhen reporting bugs, please include:\n- Clear description of the issue\n- Steps to reproduce\n- Expected behavior\n- Actual behavior\n- Screenshots (if applicable)\n- Browser/OS information\n\n## 💡 Suggesting Features\n\n- Clearly describe the feature\n- Explain the use case\n- Provide examples if possible\n- Check if the feature already exists\n\n## 📖 Documentation\n\n- Update README.md for major changes\n- Add comments for complex logic\n- Keep documentation up-to-date\n\n## ✅ Pull Request Process\n\n1. Update README.md if needed\n2. Test your changes thoroughly\n3. Follow code style guidelines\n4. Provide clear PR description\n5. Link related issues if applicable\n6. Be responsive to feedback\n\n## 📞 Questions?\n\nFeel free to open an issue or contact:\n- **Email:** marc.schuettforth@outlook.com\n- **GitHub:** [@Forthy25](https://github.com/Forthy25)\n\n---\n\nThank you for contributing! 🙌\n
